## ✅ 日常更新流程（最常用）

### 场景一：新增单元图片

**第一步：准备图片**

1. 把图片放入 `C:\Users\31718\Desktop\待租单元可视化\unit_images\`

## 🗜️ 图片压缩（手机原图必做）

手机拍的照片通常 3-6MB，需要压缩到 **300KB 以内** 再上传，否则加载很慢。

**方法：在 PowerShell 里运行以下脚本**

```python
# 先把以下内容保存为 compress.py，放在桌面
# 然后运行：python C:\Users\31718\Desktop\compress.py
```

或者直接在 PowerShell 运行（一次性压缩所有超300KB的图片）：

```
python -c "
from PIL import Image
from pathlib import Path

img_dir = Path(r'C:\Users\31718\Desktop\待租单元可视化\unit_images')

for f in img_dir.glob('*.jpg'):
    size = f.stat().st_size
    if size > 300 * 1024:
        before = size // 1024
        img = Image.open(f)
        w, h = img.size
        if max(w, h) > 1600:
            ratio = 1600 / max(w, h)
            img = img.resize((int(w*ratio), int(h*ratio)), Image.LANCZOS)
        if img.mode in ('RGBA', 'P'):
            img = img.convert('RGB')
        img.save(f, 'JPEG', quality=72, optimize=True)
        after = f.stat().st_size // 1024
        print(f.name + ': ' + str(before) + 'KB -> ' + str(after) + 'KB')

print('压缩完成')
"
```

> 💡 PNG 文件通常本身就不大，不需要压缩。

## 🏗️ 图片命名规则

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 单元图片 | `单元号-序号.jpg` 或 `.png` | `504-1.png`, `1207-3.jpg` |
| 楼层平面图 | `floor_楼层号.jpg` | `floor_4.jpg`, `floor_13A.jpg` |

**注意：**
- 序号从 1 开始，不能跳号（有 1、2、3，不能直接 1、3）
- 同一单元的第一张图（`-1`）通常是户型图（PNG），后续是实景图（JPG）
- 大小写无所谓，脚本会自动识别

## 🆘 遇到问题怎么办

### 问题1：运行脚本出错

检查：
- Python 是否安装？运行 `python --version`
- Pillow 库是否安装？运行 `pip install Pillow`

### 问题2：GitHub Desktop 看不到变更

- 确认步骤三的图片同步命令已运行
- 确认构建脚本已成功运行（index.html 有更新时间变化）

### 问题3：推送后网页没更新

- 等 2-3 分钟，GitHub Pages 有延迟
- 手机清一下缓存（微信里长按链接 → 刷新）
- 在浏览器地址栏加 `?v=随机数字` 强制刷新，如：`https://wangmeiyi411830.github.io/RCC/?v=2`

### 问题4：忘了怎么操作，需要AI协助

把这个文件内容发给AI助手，说"帮我更新电子图册"，AI 就能立刻了解全部背景，无缝协助你。
